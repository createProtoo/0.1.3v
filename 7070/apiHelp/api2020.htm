前端订单请求md5签名校验格式【外部订单号+订单金额+商户号+商户KEY】格式示例如下:  Sign

参数名ASCII码从小到大排序（字典序），使用URL键值对的格式（key1=value1&key2=value2…）拼接成字符串

$Sign='OrderMoney=58.00&OrderNo=1515420618342801&PosUid=699621&Key=298eba11f1f4a1564c4726a472ece4e7’

注意：如果觉得字典排序麻烦，该演示为最终字典顺序，可直接这么排序

md5($Sign)

// 签名步骤一 ：按字典序排序

// 签名步骤二 ：在 string 后加入 Key

// 签名步骤三 ：md5 加密

 

 

 

 

 

 

异步回调通知支付结果（POST回调）application/x-www.form-urlencoded数据类型

判断验签是否合法，32位签名格式下面是签名格式

【外部订单号+系统订单号+订单金额+商户号+商户密钥】

回调签名格式方法同上【字符示例】注意该示例为实际排序:

$Sign='OrderID=20191215154010295935&OrderMoney=1.00&OrderNo=2019666666660888&PosUid=999999&Key=999999'

md5($Sign)

 

商户方接收到参数后，请务必对OrderMoney订单金额,比对是否与发起的金额一致POST 请打印调试回调数据,回调返回:冲值成功后 返回值: success 失败返回值: failed
